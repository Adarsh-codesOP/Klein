# Security Policy

## Supported Versions

Klein is currently under active development. Security fixes will only be applied to the latest stable version.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅         |
| Older releases | ❌         |

Users are strongly encouraged to always use the **latest release** to receive security updates and bug fixes.

---

## Reporting a Vulnerability

If you discover a security vulnerability in **Klein**, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please report it privately:

* Open a **GitHub Security Advisory**
* Or email the maintainer: **[adarshas.work@gmail.com](mailto:adarshas.work@gmail.com)**

Include the following details in your report:

* Description of the vulnerability
* Steps to reproduce the issue
* Possible impact or exploit scenario
* Suggested fix (if available)

---

## Response Process

Once a vulnerability is reported:

1. The maintainer will acknowledge the report within **48 hours**.
2. The issue will be investigated and validated.
3. A fix will be developed and released as soon as possible.
4. Security updates will be announced in the **GitHub releases page**.

If the vulnerability is accepted, a patch will be released and credit may be given to the reporter (if desired).

---

## Responsible Disclosure

Please allow time to investigate and patch vulnerabilities before publicly disclosing them. Responsible disclosure helps protect users of the project.

Thank you for helping keep **Klein** secure.
